# 日志系统设计

## 什么是日志？

我们使用**日志**记录整个系统的运行情况。包含以下情况：

+ HTTP请求
+ SQL请求
+ 错误
+ 第三方接口
+ ...



## 目的

+ 方便调试，更快的帮我们定位问题



## 使用场景

+ 本地开发
+ 线上



## 等级

+ debug
+ info
+ warn
+ error



## 库

1. `tracing`

   一个有范围的、结构化的日志和诊断系统

   异步编程中使用传统的日志是存在一些问题的，因为异步任务执行起来没有确定的顺序，输出的时候没有确定的逻辑顺序。

   `tracing`引入了`span`概念，一个span代表了从开始到结束的一个时间段，在此期间的信息都可以记录。
